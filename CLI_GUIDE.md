# QED v0.2 Administrator CLI Guide

This guide describes how to use the command-line interface (CLI) to manage user profiles and invitation tokens for the QED application.

## Prerequisites

- Access to the server terminal.
- Node.js installed on the system.
- Commands should be run from the project root directory (`qed/`).

## Basic Usage

Run the CLI using Node.js:

```bash
node server/cli.js <command> [arguments]
```

## Available Commands

### 1. Invitation Token Management

New profiles require a valid invitation token for creation via the web UI.

#### Generate a Token
Generates a unique 11-character token starting with `%`.
```bash
node server/cli.js token
```
*Output example: `Generated Token: %ABC123DefG`*

#### Clear All Tokens
Deletes all currently pending invitation codes.
```bash
node server/cli.js clear-tokens
```

---

### 2. Profile Management

#### List All Profiles
Lists all currently active profile directories.
```bash
node server/cli.js list
```

#### Create a Profile (Admin Bypass)
Directly creates a profile directory without needing an invitation token.
```bash
node server/cli.js create <username>
```

#### Remove a Profile (Backup)
Renames a profile folder to `[name]_del`. **Warning**: If a backup already exists for this name, it will be overwritten by the new backup.
```bash
node server/cli.js remove <username>
```

#### Rename a Profile
Changes the directory name of an existing profile.
```bash
node server/cli.js rename <old_name> <new_name>
```

#### Reset a Profile
Wipes all learning history, progress, stars, and configuration data for a user, resetting them to factory defaults.
```bash
node server/cli.js reset <username>
```

---

## Token Storage
Pending tokens are stored in `tokens.json` in the project root. This file is automatically created and updated by the CLI and the server. Do not edit this file manually unless you know what you are doing.
